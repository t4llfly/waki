# Waki

Многофункциональный и высокопроизводительный музыкальный Discord-бот, написанный на Python. Он использует Lavalink для стабильного и качественного воспроизведения звука, а также включает интерактивный пользовательский интерфейс для управления музыкой прямо из текстовых каналов Discord.

## Особенности

- **Высококачественный звук:** Работает на базе Lavalink и FFmpeg для плавной потоковой передачи музыки.
- **Интерактивное управление:** Включает современный пользовательский интерфейс Discord (`MusicControlView`) с кнопками для быстрого управления: Пауза, Продолжить, Пропустить и Стоп.
- **Управление очередью:** Удобный просмотр и управление музыкальной очередью вашего сервера.
- **Управление громкостью:** Изменение громкости воспроизведения "на лету".
- **Готовность к Docker:** Полностью контейнеризирован с минимальным потреблением ресурсов (лимиты CPU и памяти уже настроены).

## Команды

Бот поддерживает следующие команды (как правило, в виде слэш-команд, в зависимости от вашей настройки):

- `/join` - Заставляет бота подключиться к вашему активному голосовому каналу.
- `/play <запрос/ссылка>` - Воспроизводит трек или добавляет его в очередь.
- `/skip` - Пропускает текущий играющий трек.
- `/queue` - Отображает текущую музыкальную очередь.
- `/stop` - Останавливает воспроизведение музыки и очищает очередь.
- `/leave` - Отключает бота от голосового канала.
- `/volume <уровень>` - Изменяет громкость аудиоплеера.

## Требования (Prerequisites)

- [Docker](https://docs.docker.com/get-docker/) и [Docker Compose](https://docs.docker.com/compose/install/)
- [Токен Discord бота](https://discord.com/developers/applications)
- Запущенный сервер [Lavalink](https://lavalink.dev/) (боту необходим Lavalink для обработки и трансляции звука)

## Установка и запуск

1. **Клонируйте репозиторий:**
   ```bash
   git clone <ссылка-на-репозиторий>
   cd musicbot
   ```

2. **Настройте переменные окружения:**
   Создайте файл `.env` в корневой директории проекта. Вам нужно будет указать токен вашего Discord бота и данные для подключения к Lavalink. Пример:
   ```env
   DISCORD_TOKEN=ваш_токен_discord_бота
   LAVALINK_HOST=ваш_хост_lavalink
   LAVALINK_PORT=2333
   LAVALINK_PASSWORD=youshallnotpass
   ```

3. **Запустите бота:**
   Используйте Docker Compose для сборки и запуска бота в фоновом режиме:
   ```bash
   docker-compose up -d --build
   ```

## Управление ботом

- **Просмотр логов:** 
  ```bash
  docker logs -f waki_music_bot
  ```
- **Остановка бота:**
  ```bash
  docker-compose down
  ```
- **Перезапуск бота:**
  ```bash
  docker-compose restart
  ```

## Заметки об архитектуре

- **`bot.py`**: Основная логика приложения. Управляет взаимодействием с API Discord, подключениями к узлам Lavalink, состоянием плеера (`MusicPlayer`) и элементами интерфейса (`MusicControlView`).
- **`Dockerfile`**: Собирает легковесный образ на базе Python 3.12, устанавливает системные зависимости (например, `ffmpeg`) и Python-пакеты.
- **`docker-compose.yml`**: Управляет жизненным циклом контейнера, внедряет файл `.env` и устанавливает строгие лимиты ресурсов (0.5 CPU, 512MB RAM), чтобы бот работал эффективно.
